Installation

After downloading and extracting template package you will find template folders and files.

In order to start working with our template, all you have to do is copy chosen template files on your FTP server.

For further template modifications please head to Customization section.

Files structure

We do our best while making all files for Campfire. Every file (HTML, CSS, JS) that was created by us has clear code, wrapped with comments - all this for your convenience.

Template files structure:

  • css - CSS styles folder
  • fonts - webfont icons folder
  • img - folder for images
  • js - JavaScript files folder
  • php - PHP contact script folder
  • twitter - Twitter script files
  • index.html - main HTML file

CSS

Main CSS file is style.css which has been divided into 18 sections:

  • Import section - place where you include other CSS files (webfonts, your own stylesheets)
  • Basic styles - basic template styles and classes
  • Header section - header module
  • Image right section - product showcase (right) module
  • image left section - product showcase (left) module
  • button section - call to action button module
  • Features section #1 - features with icons and texts module #1
  • Features section #2 - features with icons and texts module #2
  • Pricing section - price plans module
  • Panels section - carousel list module
  • News section - news with blog links module
  • Comment section - comment / quote module with image
  • Info box section - two column module with button
  • Team section - team members module
  • Twitter section - Twitter feed module
  • Contact section - contact form module
  • Footer section - footer menu module
  • Breakpoints - additional media queries for responsive CSS styles

JavaScript

All JavaScripts files are placed in js folder and have to be included at the bottom of the HTML file (before </body> tag).

Our custom JS file is called lj-campfire.js and must be placed right after all other JS files.

In lj-campfire.js includes other scripts calls and our JS functions made for Campfire.

Fonts

Campfire uses Google Webfonts.

All these fonts are imported in style.css file in section /* 1. IMPORT SECTION */. Here is an example code:

@import url("https://fonts.googleapis.com/css?family=Roboto:400,700,300&subset=latin,latin-ext");
@import url("https://fonts.googleapis.com/css?family=Montserrat:400");

If you want to import new font from Google Webfont library, don't forget to change property in CSS style: font-family with correct font name.

General settings

Preloader

Preloader will hide contents of your site until it's fully loaded. To change default loading animation please open img folder and override preloader.svg with your own file (you can use GIF or other image types). If you prefer static image icon over animation preloader, place your image please in img folder, then head over to css folder and edit style.css file. Change image file path accordingly as shown below:

.preloader {
  background: #fff url('../img/preloader.svg') no-repeat center center;
  .
  .
  .
}

By editing background property you can change background color.

You can use different preloader SVG images from here: http://samherbert.net/svg-loaders/.

In case you want to turn off preloader, please edit index.html and remove code shown below:

<!-- PRELOADER -->
<div class="lj-preloader"></div>
<!-- /PRELOADER -->

Then open lj-campfire.js in js folder and remove code shown below:

// preloader
$(window).load(function() {
  $(".preloader").delay(250).fadeOut(500);
});

Webfont icons

Campfire uses Ionicons as an webfont icon pack.

Ionicons files are placed in folders fonts and css (CSS file ionicons.min.css).

To properly use Ionicons, add icon's class name to your HTML tag. Example: <i class="ion-android-checkmark-circle"></i>.

Header section

Background image

You can set a background image using Backstretch plugin which function is initialized in lj-campfire.js:

// backstretch
$("header").backstretch("img/header.jpg");

Simply change background image file header.jpg in img folder.

Logo

Logo image must be placed in img folder and named logo.png.

<!-- LOGO -->
<div class="col-sm-4 logo">
  <img src="img/logo.png" alt="">
</div>
<!-- /LOGO -->

Default logo image resolution should be 2x bigger than set in CSS:

.logo img {
  display: block;
  width: 200px;
  height: 80px;
  margin-top: 10px;
}

Logo image should have resolution 400x160 pixels. You can change default resolution to desired values.

In order to do this you need to edit style.css file in css folder. Look for .logo class as specified below, and change it's width and height values.

Menu

Menu in header is used to scroll to selected module from the template. Here is an HTML code:

<!-- MENU -->
<div class="col-sm-8 menu">
  <ul>
    <li><a href="#explore" class="scroll">Explore</a></li>
    <li><a href="#features" class="scroll">Showcase</a></li>
    <li><a href="#services" class="scroll">Services</a></li>
    <li class="separator"></li>
    <li><a href="#pricing" class="bordered scroll">Plans&nbsp;&amp;&nbsp;Pricing</a></li>
  </ul>
</div>
<!-- /MENU -->

It uses a href values from links in menu which must be the same as id of the module, for example: pricing plans module has id pricing, so link in menu should be: <a href="#pricing" class="bordered scroll">Plans&nbsp;&amp;&nbsp;Pricing</a>. There is also scroll class added to each link that have scroll animation functionality.

Twitter

This section contains Twitter feed with title:

You can find this section in index.html file marked as: <!-- TWITTER SECTION -->.

Twitter settings can be found in lj-campfire.js:

// Tweetie
$('.lj-twitter').twittie({
  username: 'lumberjacksnews',
  count: 3,
  dateFormat: '%d %b %Y',
  hideReplies: true,
  template: '<p>{{tweet}}</p><span>{{date}}</span>',
  apiPath: 'twitter/api/tweet.php'
});

Please remember to change username to your desired Twitter user.

New Twitter API requires oAuth Token Key, so it's three step process. First, you need consumer and consumer secret keys. Get them from dev.twitter.com/apps. Edit twitter/api/config.php file and replace variables with your Consumer and oAuth Keys:

// Consumer Key
define('CONSUMER_KEY', 'CONSUMER_KEY_HERE');
define('CONSUMER_SECRET', 'CONSUMER_SECRET_HERE');

// User Access Token
define('ACCESS_TOKEN', 'ACCESS_TOKEN_HERE');
define('ACCESS_SECRET', 'ACCESS_SECRET_HERE');

All CSS styles you can find in style.css in section called /* 15. TWITTER SECTION */.

Contact section

You can use AJAX contact form which contains name, e-mail, phone number and message.

You can find this section in index.html file marked as: <!-- CONTACT SECTION -->.

Validation script is included in our custom JS file lj-campfire.js.

In php folder you can find file called send-contact.php. All PHP code for handling form submitting proccess can be put in this file. The variable that you can use in PHP functions is $_POST['contact_name'] to get a value of entered name, $_POST['contact_email'] to get entered e-mail, $_POST['contact_phone'] to get entered phone number and $_POST['contact_message'] to get message content.

For those who are not familiar with PHP programming we've prepared ready-to-use basic example - sending contact form data to your e-mail:

// Please put your PHP code here
//
// You can use entered name variable as $_POST['contact_name'],
// e-mail variable as $_POST['contact_email'],
// phone variable as $_POST['contact_phone'],
// message variable as $_POST['contact_message']

// Please insert here your email address: 
$to      = 'PUT_YOUR_EMAIL_ADDRESS_HERE';

$subject = 'Campfire / Message from '.$_POST['contact_name'];
$message = $_POST['contact_message']. "\r\nE-mail: " .$_POST['contact_email']. "\r\nPhone: " .$_POST['contact_phone'];
$headers = 'From: '.$_POST['contact_email']. "\r\n" .
    'Reply-To: '.$_POST['contact_email']. "\r\n" .
    'X-Mailer: PHP/' . phpversion();

mail($to, $subject, $message, $headers);

Please remember to change $to variable to your email address.

All CSS styles you can find in style.css in section called /* 16. CONTACT SECTION */.

Q: Why images included in ThemeForest's .zip file are blurred?
A: The answer is really simple. We have no rights to sell them. They're used only for a preview on our demo previews. But don't worry! We have included in credits the URLs to all images/photos/videos in Credits section.

Q: My JavaScript script doesn't work, why?
A: Please check that you've included it properly, check filename in folder and HTML file. Also please make sure that you have properly call it in lj-campfire.js file.

Q: How can I change the Favicon?
A: In folder img you can find image called favicon.png. All you have to do is to make a new image (we recommend a 32x32 px .png) and replace Campfire's default favicon.
